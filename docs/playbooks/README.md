# Shifty QE Playbooks

This directory contains comprehensive Quality Engineering playbooks aligned with the MAMOS (Mindset, Architecture, Methods, Organization, Sustainability) model.

## Persona-Specific Playbooks

| Persona | Playbook | Description |
|---------|----------|-------------|
| Product Owner/Manager | [product-owner.md](./product-owner.md) | Release readiness, ROI metrics, incident prevention |
| Designer | [designer.md](./designer.md) | Visual quality, UX regressions, accessibility |
| QA/SDET | [qa-sdet.md](./qa-sdet.md) | Manual testing, automation, defect insights |
| Developer | [developer.md](./developer.md) | SDK integration, test generation, CI pipeline health |
| Engineering Manager | [engineering-manager.md](./engineering-manager.md) | DORA/SPACE dashboards, ROI analytics |

## MAMOS Framework

### Mindset
- Quality is everyone's responsibility
- Shift-left testing practices
- Continuous improvement culture
- Data-driven decision making

### Architecture
- Telemetry-first design
- Quality data lake
- Retraining pipelines
- Multi-tenant isolation

### Methods
- Test modality coverage (functional, performance, security, accessibility)
- Automation-first with manual testing for exploratory
- Production feedback loops
- Chaos engineering

### Organization
- Persona-aware dashboards
- Cross-functional collaboration
- HITL (Human-in-the-Loop) integration
- QE governance policies

### Sustainability
- Weekly retraining cycles
- Automated quality gates
- Technical debt tracking
- Operational runbooks

## Getting Started

1. Identify your persona
2. Follow the relevant playbook
3. Use the SDK and tools described
4. Monitor your quality metrics
5. Continuously improve based on insights
